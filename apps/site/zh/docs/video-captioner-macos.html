VideoCaptioner macOS 上不可用

最后更新：2026 年 5 月 6 日 · 分类：问题排查

本文为占位示例，作为后续撰写文档的模板。 实际现象与解决步骤会随 SMM 版本更新而变化。

现象

在 macOS 上打开 设置 → 字幕 时，自动字幕 （VideoCaptioner） 选项呈灰色，并提示类似：

VideoCaptioner 当前仅在 Windows 上可用。

原因

VideoCaptioner 依赖一个第三方语音转写引擎，目前仅打包了 Windows 版本，macOS 与 Linux 的二进制还在跟进中，尚未默认随应用一起分发。

解决方法

  1. 使用远程转写服务。如果你有 OpenAI 兼容的 Whisper 接口，可在 设置 → AI → 转写 里配置远程地址，SMM 会优先使用它。
  2. 手动生成字幕。使用 whisper.cpp 等工具在终端中生成 .srt 文件，把它放在与视频文件同一目录，SMM 会自动 识别为关联字幕。
  3. 在 Windows 虚拟机中运行 SMM。如果你只是偶尔需要 生成字幕，并希望使用内置工作流，这是最简单的方式。

在批量处理之前，请先确保远程转写服务可访问，避免大量任务失败 之后还要手动重试。

关注 macOS 原生支持

macOS 原生支持的进度在 GitHub 上跟踪。如果你希望在 macOS 版本可用 时第一时间收到通知，可以订阅相关 Issue：

  • GitHub Issues —— 搜索 VideoCaptioner macOS。
