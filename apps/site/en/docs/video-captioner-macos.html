VideoCaptioner is not available on macOS

Last updated: May 6, 2026 · Category: Troubleshooting

This is a placeholder article used as a template for future docs. The actual issue and steps may change as SMM evolves.

Symptom

When you open Settings → Subtitles in Simple Media Manager on macOS, the VideoCaptioner auto-subtitle feature appears greyed out, and you see a message like:

VideoCaptioner is currently only available on Windows.

Cause

VideoCaptioner relies on a third-party transcription engine that, for now, only ships with a Windows build. macOS and Linux packages are tracked but not yet bundled by default.

Workarounds

  1. Use a remote transcription endpoint. If you have access to an OpenAI-compatible Whisper endpoint, configure it in Settings → AI → Transcription and SMM will use it instead.
  2. Generate subtitles manually. Run whisper.cpp or any compatible tool from the terminal and place the resulting .srt next to your video file. SMM will pick it up automatically.
  3. Run SMM in a Windows VM if you only need subtitles occasionally and prefer the built-in workflow.

Make sure your remote transcription endpoint is reachable before triggering large batches — failed jobs will need to be retried manually.

Tracking native macOS support

Native macOS support is tracked on GitHub. Please subscribe to the issue if you’d like to be notified when a build is available:

  • GitHub Issues — search for VideoCaptioner macOS.
